✅ OK
Long-form article / report

🟡 CARE
Documentation page
OK ma reduce stagger a 30-40ms. User skimm-reads, non vuole aspettare.

Stagger su pochi item = penalty perception. Just snap-render.

❌ NO

❌ NO
Search results

05.E

Anti-pattern banned

scroll patterns vietati · accessibility + UX violations
Re-animate ogni scroll
❌ Re-animate on re-scroll

Loop on enter+exit viewport

Distrae continuamente. User non riesce mai a "fissare" la sezione. One-shot rule.
Flicker 0↔1
❌ Flicker on enter

Opacity flash 0→1 senza transition

User percepisce come "rendering bug". Use opacity transition 300ms ease-out.
Big translateY 80px
❌ Excessive translateY

TranslateY ≥ 32px

Layout shift forte. User vede "sezione dal nulla". Use ≤ 8-12px (subtle).
05.F

Integration code

CSS + IntersectionObserver · React hook · Tailwind motion-safe
/* CSS — reveal initial state + final via .in class */ .reveal { opacity: 0; transform: translateY(8px); transition: opacity 300ms cubic-bezier(0.16, 1, 0.3, 1), transform 300ms cubic-bezier(0.16, 1, 0.3, 1); } .reveal.in { opacity: 1; transform: translateY(0); } /* Stagger via CSS variable --i (set inline on each child) */ .stagger > .reveal { transition-delay: calc(var(--i, 0) * 60ms); } /* Vanilla JS — IntersectionObserver one-shot */ const io = new IntersectionObserver((entries) => { entries.forEach(e => { if (e.isIntersecting && !e.target.dataset.revealed) { e.target.dataset.revealed = '1'; e.target.classList.add('in'); } }); }, { threshold: 0.15 }); document.querySelectorAll('.reveal').forEach(el => io.observe(el)); @media (prefers-reduced-motion: reduce) { .reveal { opacity: 1 !important; transform: none !important; transition: none !important; } }
// hooks/useRevealOnScroll.ts 'use client'; import { useEffect, useRef, useState } from 'react'; export function useRevealOnScroll<T extends Element>(threshold = 0.15) { const [revealed, setRevealed] = useState(false); const ref = useRef<T>(null); useEffect(() => { const el = ref.current; if (!el || revealed) return; const io = new IntersectionObserver( ([entry]) => { if (entry.isIntersecting) { setRevealed(true); io.disconnect(); } }, { threshold } ); io.observe(el); return () => io.disconnect(); }, [revealed, threshold]); return [revealed, ref] as [boolean, React.RefObject<T>]; } // components/RevealSection.tsx import { useReducedMotion } from 'framer-motion'; export function RevealSection({ children, index = 0 }) { const [revealed, ref] = useRevealOnScroll<HTMLDivElement>(); const reduced = useReducedMotion(); return ( <div ref={ref} className={`reveal ${revealed || reduced ? 'in' : ''}`} style={{ '--i': index } as React.CSSProperties}> {children} </div> ); }
// tailwind.config.ts module.exports = { theme: { extend: { keyframes: { 'fade-up': { '0%': { opacity: '0', transform: 'translateY(8px)' }, '100%': { opacity: '1', transform: 'translateY(0)' }, }, }, animation: { 'fade-up': 'fade-up 300ms cubic-bezier(0.16,1,0.3,1) forwards', }, }, }, }; {/* Usage — JS adds .in class on viewport enter (one-shot) */} <section className="opacity-0 motion-safe:[&.in]:animate-fade-up" ... /> {/* For stagger, use inline style with --i and arbitrary delay */} <div className="motion-safe:[&.in]:animate-fade-up [animation-delay:calc(var(--i)*60ms)]" style={{ '--i': 2 }} />
