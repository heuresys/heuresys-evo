03.C

Use case map

graph viz ✅ · liste piatte ❌ · pattern dipende dalla densità relazioni
✅ OK
KG topology viz
Pattern canonical. Node hover rivela vicinato semantico (1-hop).
✅ OK
Network/cluster diagram
Es. tenant fleet, integrazioni, flussi dati. Edges focus aiuta drill.
✅ OK
Org chart interactive
Hover su persona evidenzia manager + reports diretti. Stesso pattern.
🟡 CARE
Treeview gerarchica
OK se nodi multi-relazione. Per parent-child semplice basta highlight bg.
❌ NO
Liste piatte (table rows)
Use simple bg-color hover. Scale + blur sproporzionato per row 32-48px.
❌ NO
Card grid statici
Use card-lift pattern (translateY -2px + border-color). No scale-up.
❌ NO
Bar chart / heatmap cells
Use border 2px + tooltip. Scale rompe griglia, distrugge comparabilità.
🟡 CARE
Sparkline points
OK scale 1.4 sul punto attivo + tooltip. Il pattern qui è data-point hover, non graph.
03.D

Alternatives compare

hover sui nodi delle 4 mini-grafici · vedi differenza scale
No motion
No hover effect
scale: 1 · no transform
Nessun feedback hover. User non capisce se nodo è cliccabile.
Subtle
Scale 1.05 only
scale: 1.05 · no glow
Troppo timido. Utente lo vede solo se sta già guardando il nodo. Non guida l'attenzione.
★ Recommended
Scale 1.18 + drop-shadow
scale: 1.18 · blur: 12px · 150ms
Visible al periphery vision. Glow rinforza senza esagerare. Sweet spot.
Too much
Scale 1.4 + huge glow
scale: 1.4 · blur: 24px
Layout shift visibile, copre nodi vicini. Pattern videogame, non analytics.
03.E

Anti-pattern banned

cosa NON fare nei graph viz · attention thieves
❌ Orbit infinito

Particle ring around node

Anti-slop reject (DESIGN.md Section 7). Distrazione continua, off-brand sci-fi. Use static halo.
❌ Pulse waves

Expanding shockwave loop

Sovraccarica significato. Es. "live" o "incoming data". Use semantic dot static.
❌ Magnetic jitter

Auto-translate ping-pong

Layout fisico instabile. User non riesce a click. Fa sembrare il nodo "bug rendering".
03.F

Integration code

CSS + SVG · React + adjacency map · D3 force layout
/* CSS — pure hover via :hover (no JS for adjacency) */ .kg-svg .node { cursor: pointer; transition: transform 150ms cubic-bezier(0.16, 1, 0.3, 1), filter 150ms cubic-bezier(0.16, 1, 0.3, 1); transform-box: fill-box; transform-origin: center; } .kg-svg .node:hover { transform: scale(1.18); filter: drop-shadow(0 0 12px currentColor); } .kg-svg .edge { transition: opacity 150ms ease-out; opacity: 0.6; } /* For full adjacency dim/highlight, use JS (see React tab) */
// components/KGCanvas.tsx — full adjacency hover 'use client'; import { useState, useMemo } from 'react'; type Node = { id: string; cx: number; cy: number; r: number; color: string }; type Edge = { from: string; to: string }; export function KGCanvas({ nodes, edges }: Props) { const [hoveredId, setHoveredId] = useState<string | null>(null); const adjacency = useMemo(() => { const map = new Map<string, Set<string>>(); edges.forEach(e => { map.set(e.from, (map.get(e.from) ?? new Set()).add(e.to)); map.set(e.to, (map.get(e.to) ?? new Set()).add(e.from)); }); return map; }, [edges]); const related = hoveredId ? adjacency.get(hoveredId) ?? new Set() : new Set(); return ( <svg className="kg-svg" data-hovering={!!hoveredId}> {edges.map(e => ( <line className={`edge ${e.from === hoveredId || e.to === hoveredId ? 'related' : ''}`} ... /> ))} {nodes.map(n => ( <circle className={`node ${n.id === hoveredId ? 'active' : related.has(n.id) ? 'related' : ''}`} onMouseEnter={() => setHoveredId(n.id)} onMouseLeave={() => setHoveredId(null)} ... /> ))} </svg> ); }
// hooks/useForceLayout.ts — D3 force-directed positioning import * as d3 from 'd3-force'; import { useEffect, useState } from 'react'; export function useForceLayout(nodes, edges) { const [pos, setPos] = useState(nodes); useEffect(() => { const sim = d3.forceSimulation(nodes) .force('link', d3.forceLink(edges).id((d: any) => d.id).distance(120)) .force('charge', d3.forceManyBody().strength(-300)) .force('center', d3.forceCenter(400, 210)) .on('tick', () => setPos([...nodes])); return () => sim.stop(); }, [nodes, edges]); return pos; } // Combined with KGCanvas hover, you get force layout + interactive adjacency. // Hover transitions are CSS-driven (150ms ease-out). D3 only positions.
