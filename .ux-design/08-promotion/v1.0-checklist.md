## 2 · Pre-flight checks tecnici per categoria

### 2.1 Design tokens JSON → packages/ui/design-tokens/

**Goal**: i token JSON in `.ux-design/05-theme-variants/` diventano consumibili da `@heuresys/ui` per build pipeline cross-platform.

**Workflow**:

```bash
# 1. Crea directory target
mkdir -p packages/ui/design-tokens

# 2. Copia JSON SoT
cp .ux-design/05-theme-variants/tokens-*.json packages/ui/design-tokens/

# 3. Aggiungi Style Dictionary v4 build script (package.json)
npm install --save-dev style-dictionary --workspace=packages/ui

# 4. Crea config sd.config.js in packages/ui/
# 5. npm run build:tokens generate CSS/JS/Swift export

# 6. Verifica round-trip: tokens-dark.json → tokens.css → match active-theme.css
```

**Acceptance**:

- [ ] JSON files mirror `.ux-design/05-theme-variants/*` byte-by-byte (no drift)
- [ ] `npm run build:tokens` genera CSS variables aligned a `active-theme.css`
- [ ] Sync direction documented: `.ux-design/` resta SoT canonical, `packages/ui/design-tokens/` è copy production
- [ ] CI step aggiunto: pre-commit check che `.ux-design/05-theme-variants/*.json` ≡ `packages/ui/design-tokens/*.json`

### 2.6 next/font integration

**Goal**: self-host i 3 font stack (Exo 2 · Inter · JetBrains Mono) via `next/font/google`. Zero CLS, zero external request.

**Workflow** (`services/app/src/app/layout.tsx`):

```tsx
import { Exo_2, Inter, JetBrains_Mono } from 'next/font/google';

const exo2 = Exo_2({
  subsets: ['latin'],
  weight: ['400', '500', '700'],
  variable: '--font-exo2',
  display: 'swap',
});

const inter = Inter({
  subsets: ['latin'],
  weight: ['400', '500', '600', '700', '800'],
  variable: '--font-inter',
  display: 'swap',
});

const jetbrains = JetBrains_Mono({
  subsets: ['latin'],
  weight: ['400', '500', '600', '700'],
  variable: '--font-jetbrains',
  display: 'swap',
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="it" className={`${exo2.variable} ${inter.variable} ${jetbrains.variable}`}>
      <body>{children}</body>
    </html>
  );
}
```

**Acceptance**:

- [ ] Variabili CSS `--font-exo2`, `--font-inter`, `--font-jetbrains` disponibili in tutto il bundle
- [ ] `active-theme.css` `@theme inline` referenzia le variabili (già configurato linea 132-134)
- [ ] Zero `<link href="https://fonts.googleapis.com">` nel HTML output (no external CDN)
- [ ] Lighthouse "fonts" score 100/100 · CLS < 0.05

## 3 · Workflow integration con `/studio:bootstrap`

Per ogni asset HTML mockup → React route, usare la skill `studio` (vedi `.claude/skills/studio/SKILL.md`):

```
/studio:bootstrap .ux-design/06-mockups/auth/login-aurora.html /login
/studio:bootstrap .ux-design/06-mockups/dashboards/cross-tenant-overview.html /dashboard/cross_tenant_overview
```

Il bootstrap genera scaffold React + copia mockup HTML in `services/app/<route>/_reference.html` come baseline · poi traduzione manuale o assistita a TSX.

Per ri-iterazione su route già promossa, usare `/studio:clone <route>` → manipola in staging → `/studio:promote` con 5-gate flow.

## 4 · Rollback plan

Ogni promotion è reversibile:

| Categoria                   | Rollback                                                                           |
| --------------------------- | ---------------------------------------------------------------------------------- |
| Design tokens JSON          | `git revert` commit di copia in `packages/ui/design-tokens/` (no impatto runtime)  |
| Logo SVG public/brand       | `git revert` (asset statici, no breaking change)                                   |
| Favicon / apple-touch       | Mantenuti in `public/` come default Next.js — rimozione = browser fallback default |
| OG image                    | `git revert` · `<meta og:image>` cade su default (text-only sharing)               |
| next/font integration       | `git revert` layout.tsx → CDN Google Fonts esterno (perf hit, no break)            |
| Motion library              | `git revert` · pattern presets restano in JSON SoT, nessun runtime dipende ancora  |
| Route promotion (`/studio`) | `/studio:restore <route> <bkp-TS>` automatic backup-restorable                     |
| Surface utility             | `git revert` su mockup commit                                                      |

## 5 · Cadence promotion (raccomandata)

**Approccio**: incremental, non bulk. Una categoria alla volta · commit + push direct main · visual smoke · move on.

Sequence raccomandata:

1. **Quick wins (1-2h)**: design tokens JSON copy + logo SVG public/brand + favicon multi-size + apple-touch-icon + og-image (asset statici, basso rischio)
2. **next/font integration (1-2h)**: refactor layout.tsx + verifica zero CLS + Lighthouse pass
3. **Motion library (2-3h)**: estrazione 5 pattern presets + utility CSS/variants TS
4. **4 surface utility (4-6h)**: 4 mockup HTML + 4 promotion via `/studio:bootstrap`
5. **Brand book v1 visivo (8-12h)**: typesetting + cover art + asset embed alta risoluzione (futuro, scope opzionale)

Total effort v1.0 promotion: ~16-25h (eseguibile in 2-3 sessioni autonomous).
