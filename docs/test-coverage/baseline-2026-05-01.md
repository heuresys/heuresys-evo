## ADR-0002 status decision (NOT promoted)

**Decision**: ADR-0002 (`docs/decisions/0002-database-testing-strategy-ci.md`) remains
in `Status: Proposed`. **Not promoted** to Accepted.

**Rationale**:
- ADR-0002 is specifically about **database integration testing in CI** via
  `@testcontainers/postgresql` + pgvector image. Its acceptance criterion is
  "the first test integration is written using this strategy" (paraphrased from
  HANDOFF Open questions).
- The 123 tests delivered in BLOCK 7 are **all unit tests** with mocked DB / mocked
  bcrypt — they do NOT exercise testcontainers, do NOT spin up a real Postgres,
  do NOT validate the lifecycle helper described in ADR-0002.
- Promoting ADR-0002 to Accepted now would falsely signal the testcontainers
  pattern is in use; in fact, no integration test has been written yet.

**RTG §9.1 task 3.8 wording** (`promote ADR-0002 da Proposed a Accepted`) appears
to conflate "first test exists" with "ADR-0002 strategy in use". This baseline
report distinguishes the two:
- ✅ First tests exist → unit test floor satisfied
- ❌ Testcontainers integration test → not yet written

**Next gate for ADR-0002 promote**: the first `*.integration.test.ts` that uses
the helper sketched in ADR-0002 (e.g. `services/api-gateway/__tests__/integration/employees.it.test.ts`).
This naturally fits in Phase 4 task 4.10 (endpoint port) where the first port
endpoint will need to validate against a real Postgres with pgvector.
