# Auto-handoff hook reference

A Stop-event hook that writes a breadcrumb to `.handoff/auto/<UTC-timestamp>.md` whenever Claude Code CLI session ends. Provides resilience against abrupt session closes (crash, ctrl+C, terminal timeout) before the user runs `/handoff`.

## Files

- `.claude/hooks/auto-handoff.sh` — the hook script (POSIX bash, runs in Git Bash on Windows)
- `.claude/settings.json` — hook registration in `hooks.Stop`
- `.handoff/auto/` — directory where breadcrumbs accumulate

## Hook script content

```bash
#!/bin/bash
# .claude/hooks/auto-handoff.sh
# Writes a breadcrumb to .handoff/auto/<timestamp>.md on every Stop event.
# Provides resilience: if session ends abruptly, next session can recover state.

set -uo pipefail

# Resolve repo root from script location
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
AUTO_DIR="$REPO_ROOT/.handoff/auto"

mkdir -p "$AUTO_DIR"

TIMESTAMP="$(date -u +%Y%m%d-%H%M%S)"
OUT="$AUTO_DIR/$TIMESTAMP.md"

# Capture state
BRANCH=$(cd "$REPO_ROOT" && git branch --show-current 2>/dev/null || echo "?")
LAST_COMMIT=$(cd "$REPO_ROOT" && git log -1 --oneline 2>/dev/null || echo "?")
GIT_STATUS=$(cd "$REPO_ROOT" && git status --short 2>/dev/null || echo "")

cat > "$OUT" <<EOF
# Auto-handoff breadcrumb — $TIMESTAMP

- timestamp_utc: $TIMESTAMP
- branch: $BRANCH
- last_commit: $LAST_COMMIT

## git status (short)

\`\`\`
$GIT_STATUS
\`\`\`

> Auto-generated by .claude/hooks/auto-handoff.sh on Stop event.
> Promote to HANDOFF.md proper at next session if state is meaningful.
EOF

# Retention: keep only the most recent N breadcrumbs (default 50, S10).
# Override via env: AUTO_HANDOFF_KEEP_LAST=N (0 disables rotation).
KEEP_LAST="${AUTO_HANDOFF_KEEP_LAST:-50}"
if [ "$KEEP_LAST" -gt 0 ]; then
  ls -1 "$AUTO_DIR"/*.md 2>/dev/null \
    | sort \
    | head -n "-$KEEP_LAST" \
    | xargs -r rm -f
fi
```

## Settings registration

Add to `.claude/settings.json`:

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/auto-handoff.sh"
          }
        ]
      }
    ]
  }
}
```

If `settings.json` already has hooks, **merge** the Stop hook with existing ones — never overwrite the file.

## Promotion at next session start

The agent (instructed by `CLAUDE.md` root §Bootstrap) at every session start:

1. Read `.handoff/HANDOFF.md`
2. Read `.handoff/PROJECT-STATE.md`
3. **Scan `.handoff/auto/` for breadcrumbs newer than `.handoff/HANDOFF.md` mtime**
4. If found: surface them to the user with a 1-line summary each, ask whether to incorporate
5. After incorporation (or skip), proceed with the standard bootstrap flow

This effectively handles the case where the user's last session ended without a clean `/handoff` — the breadcrumbs from `auto/` carry the lost state forward.

## Maintenance

- `auto/` accumulates one file per Stop event. Files are tiny (~400 bytes each).
- **Retention automatica (S10, 2026-05-04)**: il hook rimuove i file `.handoff/auto/*.md` oltre i 50 più recenti ad ogni esecuzione. Override via env `AUTO_HANDOFF_KEEP_LAST=N` (0 disabilita rotation). Filename pattern `YYYYMMDD-HHMMSS.md` ordina lessicograficamente = cronologicamente, quindi `sort | head -n -50` mantiene i 50 più recenti.
- Pre-S10: cleanup era manuale (`find .handoff/auto/ -mtime +30 -delete`). Migration nota: durante S10 first rotation 194→50 eseguita su dataset reale, backup tarball pre-rotation in `~/.handoff-archive/heuresys-evo/auto-S10-YYYYMMDDTHHMMSSZ.tar.gz`.
- La directory `.handoff/auto/` può o no essere gitignored — verifica `.gitignore` del repo. In `heuresys-evo` è **non gitignored** (i breadcrumb sono safety net distribuiti).

## Disabling

To temporarily disable: comment out the Stop hook in `.claude/settings.json` or set the matcher to a never-matching pattern. To remove permanently: delete the hook file and remove from settings.
